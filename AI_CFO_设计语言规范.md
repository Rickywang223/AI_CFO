# AI CFO 设计语言规范 v1.0

> 企业财税 AI 操作系统 · 设计原则与风格指南
> 适用场景：AI财税运营中心 / AI CFO 全系产品

---

## 一、核心设计价值观

### 1️⃣ 确定性（Certainty）
> 用户操作前就能预知结果，操作后立刻知道发生了什么。

| 场景 | 反面 | 正面 |
|:----|:----|:----|
| 点击技能卡片 | 弹出一个新页面，用户迷失 | 在右侧直接展开对话，顶部有"←返回" |
| 发送消息 | 没有 loading，用户以为没点中 | 立即显示用户气泡，AI生成中显示状态 |
| 发票认证 | 系统静默处理 | 显示"AI已自动认证 X 张，待您复核 Y 张" |

**规则：** 每一个用户操作，必须在 200ms 内给出视觉反馈。

### 2️⃣ 匹配用户心智（Match Mental Model）
> 用税务经理的语言说话，不要用工程师的语言。

| 用户说的 | 系统应该叫 |
|:--------|:----------|
| "这个月要交多少税" | 📋 申报管理（不是"纳税申报预测模块"）|
| "有哪些发票有问题" | ⚠️ 风险监控（不是"发票数据异常检测系统"）|
| "帮我写一份报告" | 📝 报告中心（不是"自动化报表生成器"）|

**规则：** 所有功能名称、按钮文案、提示语，必须能让税务经理**一眼看懂**，不需要培训。

### 3️⃣ 效率优先（Efficiency First）
> 财税工作是时间敏感的（月初认证、月中申报、月底结账），每多一步操作都是成本。

| 模式 | 做法 |
|:----|:----|
| 高频操作 | 一键完成（AI自动认证、一键申报） |
| 中频操作 | 两步完成（技能卡片→点击执行） |
| 低频操作 | 在侧栏或菜单中可找到 |

**规则：** 核心工作流 ≤ 2 步完成。超过 2 步的，说明设计有问题。

### 4️⃣ 数据可见（Data Transparency）
> 一切决策要有数据支撑，一切结果要有数据验证。

- "建议优先处理 P0 级风险" → 要有数字（3,573张不合规发票）
- "本月预估增值税 1,600万" → 要有对比（环比+5.2%，同比-8.1%）
- 每个 KPI 卡片都要有**目标值对标**

**规则：** 没有数据的结论不在 UI 中出现。

## 二、布局原则

### 2.1 黄金布局：左操作 + 右内容

```
┌─────────────────────┬──────────────────────────────────────┐
│ 侧栏 (220-280px)    │ 主内容区 (flex: 1)                   │
│                     │                                      │
│ 风险提醒（紧凑）     │ 驾驶舱：KPI卡片 + 图表 + 列表        │
│ 对话记录（优先）     │ 聊天视图：消息流 + AI回复            │
│ 配置控制（底部紧凑） │                                      │
│ 输入框（底部）       │                                      │
└─────────────────────┴──────────────────────────────────────┘
```

**原则：**
- 侧栏负责**操作入口和上下文切换**
- 主内容区负责**信息展示和任务执行**
- 底部固定区负责**模型/技能配置和输入**

### 2.2 三个固定层

| 层级 | 内容 | 行为 |
|:----|:----|:----|
| 顶部导航 | 产品名称 + 模块Tab | 固定，永远可见 |
| 底部控制 | SKILL/AGENT/模型/MCP | 固定，但紧凑 |
| 底部输入 | 消息输入 + 快捷按钮 | 固定，永远是输入入口 |

### 2.3 页面切换规则

- 点击导航Tab → 整页切换
- 点击左侧对话记录 → 右侧切换上下文
- 驾驶舱 ↔ 聊天视图 → 同页面切换，保留状态

## 三、视觉风格

### 3.1 配色

```
🔵 主色：#1a1a2e (深蓝) → 顶部导航、标题、品牌感
🟦 辅助色：#1890ff (蓝色) → 交互元素、链接、按钮
🟩 成功色：#52c41a (绿色) → 自动化、合规、正向状态
🟧 警告色：#faad14 (橙色) → 待确认、需复核、中性提醒
🔴 危险色：#f5222d (红色) → 风险预警、P0级别、不合规
⬜ 背景色：#f5f6fa (浅灰) → 页面背景
⬜ 卡片色：#ffffff (白色) → 卡片、面板
```

**规则：** 颜色使用要有语义，不能为了好看而用色。
- 绿色只用于"自动完成/合规/正向"
- 橙色只用于"需要人注意/待处理"
- 红色只用于"必须马上处理/违规"

### 3.2 字体层级

```
H1：16px / Bold → 页面标题（如"📊 驾驶舱"）
H2：14px / Semi-Bold → 模块标题（如"自动化覆盖率"）
H3：13px / Semi-Bold → 卡片标题
正文：12-13px / Regular → 内容文字
辅助：10-11px / Regular → 时间戳、标签、次要信息
数字：24-26px / Bold → KPI 数值
```

### 3.3 间距系统

```
基础间距：4px
卡片内边距：16-20px
卡片间距：12-16px
模块间距：16-20px
侧栏内边距：12px
```

**规则：** 所有间距遵循 4px 倍数，拒绝随意间距。

### 3.4 圆角

```
卡片：8px
按钮：6px
输入框：8px
标签/徽章：4px 或 12px（胶囊形）
弹窗：12px
```

## 四、交互模式

### 4.1 AI 对话交互

```
用户输入 → 蓝色气泡（右对齐）
AI 回复 → 灰色气泡（左对齐）
AI 处理中 → 显示"正在分析..."状态
AI 回复格式 → 支持：文字、表格、列表、建议
```

**规则：**
- AI 回复必须结构化（标题+要点+数据+建议）
- AI 回复结尾必须有明确的**下一步建议**
- AI 回复中所有数据必须标注来源

### 4.2 快捷操作

```
技能卡片 → 点击即执行，一步到位
快捷按钮 → 常见问题预设，一键提问
KPI 卡片 → 点击可下钻查看更多
对话记录 → 点击切换上下文
```

### 4.3 反馈机制

| 操作 | 反馈 |
|:----|:----|
| 点击 | 立即视觉反馈（背景色变化） |
| 发送消息 | 立即显示用户气泡 |
| AI 回复中 | 加载状态指示 |
| 完成 | 显示结果摘要 |
| 异常 | 红色提示 + 建议行动 |

## 五、组件规范

### 5.1 KPI 卡片

```
┌──────────────────┐
│ 📄 发票总数       │ ← 图标 + 标签
│ 1,309,454        │ ← 大数字，26px/Bold
│ 2024-2026 累计   │ ← 辅助说明
│ ● 目标 85% · 当前│ ← 目标值对标（可选）
└──────────────────┘
```

### 5.2 对话记录

```
● 进项管理分析     ← 色点 + 标题
  发票认证状态...    ← 预览摘要
```

- 色点颜色表示对话类型（蓝=进项、红=风险、绿=申报）
- 点击整行切换上下文
- 当前对话高亮（浅蓝背景）

### 5.3 技能选择器

```
🔧 SKILL  [📄 进项管理 ▼]   ← 下拉选择
🤖 AGENT  [税务助手] [合规] ← 标签切换
```

- SKILL 决定 AI 的专业领域
- AGENT 决定 AI 的角色风格
- 两者独立选择，互不影响

### 5.4 状态指示器

```
✅ 全自动（绿色背景） → AI 完全处理，无需人工
🔄 需复核（橙色背景） → AI 已处理，等待人工确认
⏳ 待处理（灰色背景） → 尚未开始处理
🔴 异常（红色背景） → 需要立即介入
```

## 六、AI 特有设计模式

### 6.1 人机分工可视化

> 用户需要知道"AI 做了什么"和"我需要做什么"

```
✅ AI 自动处理：发票认证 92% · 申报 85% · 监控 95%
⏳ 待人工复核：合规异常 3,573张 · 集采公司穿透核查
```

### 6.2 置信度指示

> AI 给出建议时，必须表示对自己答案的把握程度

```
🟢 高置信度（>90%）→ 直接显示结果，用户快速确认
🟡 中置信度（70-90%）→ 显示结果 + 建议人工复核
🔴 低置信度（<70%）→ 不自动处理，直接推送给人工
```

### 6.3 可解释性

> AI 的每一个结论都要能溯源

```
"建议优先处理P0级风险"
→ 展开：基于您当前有 3,573 张不合规发票，其中 892 张已超过90天认证期限
```

## 七、反模式（不要做）

| ❌ 不要 | ✅ 要 |
|:-------|:------|
| 弹窗/模态框进行对话 | 右侧内联对话 |
| 新页面跳转 | 同页面切换 |
| 工程师术语（"数据治理中台"）| 用户语言（"进项管理"）|
| 纯色大图/插画 | 数据图表/结构化展示 |
| 消费级动效（翻转/弹跳）| 最小化过渡（fade/无）|
| 隐藏式导航 | 所有操作可见可达 |
| 无数据支撑的结论 | 每个结论带数字佐证 |

---

## 八、一句话总结

> **AI CFO 的设计风格 = 企业级的稳重 × 数据驱动的透明 × AI 效率的锐利**
> 
> 不装酷，不卖萌，不炫技。
> 让税务经理用最少的步骤、最少的思考，完成最多的工作。
> 让他们信任这个系统，而不是觉得这个系统在给他们添乱。
